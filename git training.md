## Se kommandoer på https://git-scm.com/cheat-sheet

### Gennem disse øvelser antages at start branch hedder "main"
### Lav en branch, kald den feature-1
### Lav et par commits på feature-1
### Skift lidt frem og tilbage mellem main og feature-1 så du kan se effekten
### Prøv at ændre i den sidste commit, kommentar og kode
### Lav en rebase af feature-1, byt om på rækkefølgen af commits </br> skift evt. editor med git config --global core.editor "notepad" </br> brug evt. følgende editor https://micro-editor.github.io/
### Lav en ny branch, kald den feature-2
### Lav et par commits på feature-2, lad mindst 1 være i konflikt med filer på feature-1
### Merge feature-1 ind på main
### Rebase feature-2 på main, håndter konflikten
